# MAP vs LOAD vs スロットル — DYデミオ実走観察

DYデミオ (DBA-DY3W, ZJ-VE 1.3L, ワイヤースロットル) での実走データに基づく知見。

## 3つの指標の正体

| 指標 | ソース | 意味 |
|------|--------|------|
| **MAP** (PID 0x0B) | OBD-2 クエリ | インマニ実圧 (kPa)。唯一の物理量 |
| **スロットル** (PID 0x11) | OBD-2 クエリ | ペダル踏み量（バルブ開度ではない） |
| **LOAD** (0x201 B6) | CAN パッシブ | ECU計算値。スロットルとほぼ比例 |

### ワイヤースロットルの特性

DYデミオはペダル→ワイヤー→スロットルバルブが機械的に直結（ECU介入なし）。
ただし PID 0x11 はバルブ軸の角度ではなくペダル踏み量を反映する。
バルブは全開で物理的に止まるが、センサーの電圧レンジはその先まである。

## 実走確認: WOT域の挙動

2速/3速HOLD + 強制ロックアップで高速合流加速時に確認。

```
ペダル踏み込み →→→→→→→→→→→→→→→ ベタ踏み

MAP(0x0B):        ████████████████▌ 頭打ち（≒大気圧）
スロットル(0x11): ██████████████████████████ まだ上がる
LOAD(0x201 B6):   ██████████████████████████ まだ上がる（スロットルと比例）
加速:             ████████████████▌ 変わらず
```

- **MAPだけが頭打ち** = スロットルバルブ全開 → インマニ ≒ 大気圧の物理限界
- **スロットルとLOADは比例して上がり続ける** = 両方とも「ペダル踏み量」側の指標
- MAP頭打ち以降、ペダルを踏み増しても加速は変わらない

### MAPだけが「エンジンの真実」を語る

スロットルもLOADも「ドライバーの要求」を反映する。バルブ全開後もペダルを踏めば値が上がる。
MAPだけが「エンジンが実際に吸っている空気の状態」を反映する。

バルブ全開後のスロットル/LOADの上昇分は、AT車のキックダウン判定用ストローク。
MT車やHOLDモードではこの領域に物理的な意味はない。

## シフトダウン時の挙動

同じ車速でシフトダウンすると MAP が下がる。

- RPM上昇 → 1回転あたりの仕事が軽くなる → スロットル少量でOK → インマニ真空度が上がる
- 「エンジンが楽になった」ことがMAPに直接現れる
- スロットル/LOADではこの違いが見えにくい

## プロジェクトでの使い分け

| 用途 | 使うべき指標 | 理由 |
|------|-------------|------|
| 燃費計算 (Speed-Density) | **MAP** | 唯一の物理量。WOT域で正確 |
| エンブレ判定 | **MAP < 35 kPa** | 確実に判定可能 |
| スロットルアーク表示 | **スロットル (0x11)** | ペダル踏み量 = ドライバーの操作感と一致 |
| ECOランプ | 燃費値ベース | MAP由来の燃費計算結果を使用 |

LOADは燃費計算にもUI表示にも不適切:
- 燃費計算: WOT域で実態以上の値を返すため過大評価する
- UI表示: スロットルPIDとほぼ同じ値なので冗長
