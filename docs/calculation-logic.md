# 算出ロジック・閾値一覧

メーター表示・インジケーター・自動制御に関わるすべての計算式と閾値をまとめる。

> **凡例**
>
> - **config** = `configs/config.json` で車種ごとに設定可能
> - **ハードコード** = ソースコードに直接記述。変更するにはビルドが必要
> - 🚗 = 車種によってチューニングが必要な可能性がある値

## 1. 瞬間燃費推定

**ファイル:** `cmd/pi-obd-meter/fuel.go`

### 物理定数（ハードコード・車種非依存）

| 定数 | 値 | 説明 |
|---|---|---|
| 理論空燃比 | 14.7 | ガソリン車共通 |
| ガソリン密度 | 750 g/L | 標準的なレギュラーガソリン |
| 空気密度 | 1.225 g/L | 標準大気 (15°C, 1atm) |
| 標準大気圧 | 101.3 kPa | Speed-Density法の正規化に使用 |

### 🚗 車種チューニング値

| パラメータ | 現在値 | 設定方法 | 備考 |
|---|---|---|---|
| エンジン排気量 | 1.3 L | **config** (`engine_displacement_l`) | 燃費推定の基本パラメータ |
| 燃料レート補正係数 | 1.3 | **config** (`fuel_rate_correction`) | 理論値と実燃費の乖離を補正（暖機増量・過渡補正等） |
| アイドル燃料消費量 | 0.6 × 排気量 L/h | **排気量から導出** (`idleFuelRateCoeff * displacementL`) | 1.3L→0.78, 2.0L→1.2 L/h |
| エンブレ負荷閾値 | 5.0% | **ハードコード** (`load < 5.0`) | 走行中の極低負荷を燃料カットと判定 |
| エンブレMAP閾値 | 35.0 kPa | **ハードコード** (`engineBrakeMAPKPa`) | MAPセンサーがある場合の強い負圧による判定 |
| 燃費表示最低速度 | 10.0 km/h | **ハードコード** (`minDisplaySpeedKm`) | これ以下ではノイジーなため非表示 |
| 燃費表示上限 | 99.9 km/L | **ハードコード** (`maxDisplayKmL`) | 表示桁数の制限 |

### 計算フロー（優先順位: MAF > MAP > 負荷×RPM）

```
┌─ MAFセンサーあり ──→  fuelRate = MAF(g/s) × 3600 / (14.7 × 750)  [L/h]
│
├─ MAPセンサーあり ──→  Speed-Density法:
│   (MAFなし)            ve = MAP / 101.3  (体積効率の近似)
│                        airFlow = ve × (RPM / 2) × 排気量L / 60  [L/s]
│                        fuelRate = airFlow × 1.225 × 3600 / (14.7 × 750)  [L/h]
│
├─ 上記なし ─────→  負荷×RPM×排気量から推定:
│                        airFlow = (RPM / 2) × (負荷% / 100) × 排気量L / 60  [L/s]
│                        fuelRate = airFlow × 1.225 × 3600 / (14.7 × 750)  [L/h]
│
├─ RPM < 100 or 負荷 < 0.1% → fuelRate = 0.6 × 排気量 L/h（アイドル最低値）
│
├─ 補正係数を適用 ──→  fuelRate *= fuel_rate_correction (デフォルト 1.3)
│
└─ 燃費 = 速度(km/h) / fuelRate(L/h)  [km/L]
```

### エンブレ判定

走行中（≥ 10 km/h）のエンブレ（燃料カット）を2段階で検出:

1. **MAP ベース（優先）**: `intakeMAP < 35.0 kPa` — スロットル閉で強い負圧 = エンブレ
2. **負荷ベース（フォールバック）**: `load < 5.0%` — MAP非対応車向け

### 表示判定

| 条件 | 表示 | 戻り値 |
|---|---|---|
| speed < 0.5 かつ rpm < 100 | エンジン停止 | (0, 0) |
| fuelRate < 0.01 | 燃料カット | (-1, rate) |
| speed ≥ 10 かつ MAP < 35 kPa | エンブレ (MAP検出) | (-1, rate) |
| speed ≥ 10 かつ load < 5% | エンブレ (負荷検出) | (-1, rate) |
| speed < 10 | 低速非表示 | (0, rate) |
| 正常 | 燃費表示 | (km/L, rate) |

## 2. 速度ゲージ

**ファイル:** `web/static/js/gauge.js`

### アーク構成

- 角度: 270°アーク（-135° → +135°、7時半～4時半の位置）
- 🚗 最大値: **config** (`max_speed_kmh`, デフォルト 180)
- 目盛り: 9分割 × 5小目盛り = 45刻み（0, 20, 40, ... 180）

### 補間アニメーション（ハードコード）

| パラメータ | 値 | 説明 |
|---|---|---|
| LERP係数 | 0.35 | 毎フレームで差分の35%を追従 |
| スナップ閾値 | 0.05 | これ以下で目標値に即収束 |
| 停止閾値 | 0.01 | これ以下でアニメーション停止 |

### 🚗 速度帯カラー（ハードコード）

| 速度帯 | 色 | Hex |
|---|---|---|
| ≥ 120 km/h | 赤 | `#f44336` |
| ≥ 100 km/h | 橙 | `#ff9800` |
| ≥ 80 km/h | 黄 | `#ffeb3b` |
| ≥ 60 km/h | 薄緑 | `#69f0ae` |
| ≥ 30 km/h | 青 | `#42a5f5` |
| < 30 km/h | 灰青 | `#78909c` |

> 法定速度に基づくが、車種・用途によって閾値を変えたい場合がある。
> 例: 軽自動車なら 100 km/h 以上を赤にする等。

## 3. スロットルアーク

**ファイル:** `web/static/js/gauge.js`

速度ゲージの内側に重ねて表示。スロットル開度を視覚化する。

### 🚗 アイドル開度ベースライン

**config** (`throttle_idle_pct`, デフォルト 11.5)

> **車種依存度: 高。** ECU によってアイドル時の報告値が異なる。
> DYデミオは 11-12% 程度だが、車種によって 5-20% まで幅がある。
> config.json で車種ごとに設定可能。

生のスロットル値からベースラインを引いてゼロ始まりに正規化:

```
表示% = max(0, (生OBD値% − idle) / (max − idle) × 100)
```

### 色（ハードコード・車種非依存）

HSL色相が開度に応じて連続変化:

```
色相 = 210° × (1 − 開度% / 100)
```

| 開度 | 色相 | 色 |
|---|---|---|
| 0% | 210° | 青（巡航） |
| 50% | 105° | 緑 |
| 100% | 0° | 赤（全開） |

### 補間（ハードコード）

LERP係数 = 0.4（速度ゲージの 0.35 より速い追従）

## 4. MAP（インマニ圧）アーク

**ファイル:** `web/static/js/gauge.js`

速度ゲージの外側に表示。MAPセンサー搭載車のみ表示される。

### 表示範囲

- 最大値: 101.3 kPa（NA車の大気圧）
- 0 kPa（完全真空）から大気圧まで

### リードアウト

km/h 表示の下に `XXX kPa` 形式で数値表示。
値がない場合は `--` 表示。

### 色（ハードコード・車種非依存）

スロットルアークと同じ HSL 連続グラデーション:

```
色相 = 210° × (1 − MAP% / 100)
```

低い MAP（強い負圧 = スロットル閉）= 青、高い MAP（大気圧近く = 全開）= 赤。

### 補間

LERP係数 = 0.3

**ファイル:** `web/static/js/gauge.js`

### リードアウト

km/h 表示の下に `XX.X L/h` 形式で数値表示。

### 色（ハードコード・車種非依存）

同じ HSL 連続グラデーション。低い消費 = 青、高い消費 = 赤。

### 補間

## 7. ECO インジケーター

**ファイル:** `web/static/js/indicators.js`, `cmd/pi-obd-meter/api.go`

### 🚗 閾値

**km/L 判定（排気量から導出、速度によらず統一）:**

| 色 | 導出式 | 1.3L での値 | 2.0L での値 |
|---|---|---|---|
| 緑 | ≥ `round(20 / 排気量, 1)` km/L | ≥ 15.4 | ≥ 10.0 |
| 橙 | ≥ `round(8 / 排気量, 1)` km/L | ≥ 6.2 | ≥ 4.0 |
| 赤 | 上記未満 | < 6.2 | < 4.0 |

> 排気量が大きいほど燃費が悪いため、閾値を自動で下げる。
> `/api/config` で算出してフロントエンドに渡す。

**特殊状態:**

| 状態 | 色 | 表示 |
|---|---|---|
| エンブレ / 燃料カット (`eco < 0`) | 緑 | 平均燃費 |
| 停車・アイドル (`eco < 0.1`) | 消灯 | 平均燃費 or `--` |

## 8. 水温インジケーター

**ファイル:** `web/static/js/indicators.js`

### 🚗 温度閾値（ハードコード）

| 温度 | 色 | 意味 |
|---|---|---|
| データなし (≤ 0°C) | 消灯 | — |
| 1 - 69°C | 橙 | 暖機中 |
| 70 - 104°C | 緑 | 正常動作温度 |
| ≥ 105°C | 赤 | 過熱警告 |

> 一般的なサーモスタット開弁温度（80-90°C）を考慮した汎用的な閾値。
> ほとんどの車種でそのまま使えるが、サーモスタット温度が異なる車種では
> 暖機中の橙が長く点灯する可能性がある。

## 12. メンテナンスリマインダー

**ファイル:** `internal/maintenance/manager.go`

詳細は [configuration.md](configuration.md) を参照。

### 計算式

**距離ベース:**

```
進捗 = (現在総走行距離 − 最終リセット時距離) / 間隔km
警告 = 進捗 ≥ 警告閾値
期限切れ = 進捗 ≥ 1.0
```

**日付ベース:**

```
進捗 = 経過日数 / 間隔日数
警告 = 進捗 ≥ 警告閾値
期限切れ = 進捗 ≥ 1.0
```

同日再通知なし（1日1回まで）。

## 13. トリップ積算

**ファイル:** `internal/trip/tracker.go`

### 計算式（車種非依存）

| 項目 | 計算式 |
|---|---|
| 距離 Δ | `(speedKmh / 3600) × Δt秒` を毎サイクル加算 |
| 燃料消費 Δ | `(fuelRateLH / 3600) × Δt秒`（fuelRateLH > 0 の場合のみ） |
| 平均燃費 | `距離km / 燃料消費L`（0.05L 未満は非表示） |
| 平均速度 | `距離km / 走行時間秒 × 3600` |
| 走行時間 | speed > 1.0 km/h の累積秒数 |
| アイドル時間 | speed ≤ 1.0 km/h の累積秒数 |

- 10秒超のギャップはデータ欠損とみなしスキップ
- 状態永続化: **0.1km (100m) 走行ごと**にアトミック書き込み（距離ベース保存）

## 14. OBD ポーリング

**ファイル:** `internal/obd/pids.go`, `cmd/pi-obd-meter/app.go`

### PID パース式（OBD-2 標準・車種非依存）

| PID | 計算式 | 単位 |
|---|---|---|
| 0x0C RPM | `(A << 8 \| B) / 4` | rpm |
| 0x0D 車速 | `A` | km/h |
| 0x04 負荷 | `A × 100 / 255` | % |
| 0x11 スロットル | `A × 100 / 255` | % |
| 0x05 水温 | `A − 40` | °C |
| 0x10 MAF | `(A << 8 \| B) / 100` | g/s |
| 0x0B MAP | `A` | kPa |

### 🚗 ポーリング間隔

| 設定 | 値 | 設定方法 |
|---|---|---|
| ポーリング | 500 ms | **config** (`poll_interval_ms`, デフォルト 500) |
| 🚗 OBDプロトコル | "6" (CAN 11bit 500k) | **config** (`obd_protocol`) |

## 15. OBD スパイク除去フィルター

**ファイル:** `cmd/pi-obd-meter/filter.go`

OBD値のノイズ（ELM327通信エラーによる異常値）を除去する。

### パラメータ（ハードコード）

| パラメータ | 値 | 説明 |
|---|---|---|
| 最大変化量 | 20 | 1サイクルで許容する最大変化量 |
| 最大リジェクト回数 | 3 | 連続リジェクト後に値を強制受入 |

### ロジック

```
1. 前回値との差が最大変化量以内 → 受入
2. 差が最大変化量超 → リジェクト（前回値を維持）
3. 連続リジェクトが maxRejects 回に達したら → 強制受入（実際の急変に追従）
4. 正常値を受入した時点でリジェクトカウンタをリセット
```
